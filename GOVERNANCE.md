# Governance

## Purpose
This document defines how the GitHub Actions templates repository is maintained and reviewed.

## Maintainers
- Maintainers oversee the repository structure, template quality, and release decisions.
- Template owners should review changes to workflows they maintain.

## Review Process
- Changes to shared templates require review before merging.
- Security-related workflow changes should be reviewed carefully for permissions, secrets, and token scope.
- Breaking changes should include migration notes and, when practical, a transition period.

## Decision Making
- Minor documentation updates may be merged by maintainers.
- Changes that affect consumers broadly should be discussed in an issue or pull request.
- Disputes should be resolved by maintainers with the least disruptive path for consumers.
