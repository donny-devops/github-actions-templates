# Contributing to github-actions-templates

Thank you for your interest in contributing.

This repository exists to provide reusable GitHub Actions templates, so contributions should be focused, well-documented, and easy to adopt.

## Before you start

Please:
- Search existing issues and pull requests before opening a new one.
- Keep changes scoped to a single improvement when possible.
- Avoid introducing breaking changes unless they are clearly documented.

## How to contribute

### Report an issue
When opening an issue, include:
- A clear description of the problem or request.
- Steps to reproduce, if applicable.
- Expected and actual behavior.
- Relevant examples, logs, or screenshots.

### Submit a pull request
1. Fork the repository.
2. Create a branch for your change.
3. Make your edits.
4. Update or add documentation if needed.
5. Open a pull request with a clear description of the change.

## Quality expectations

Contributions should:
- Be accurate and minimal.
- Follow the existing style of the repository.
- Include examples when changing template behavior.
- Preserve compatibility whenever possible.

## Code review

Pull requests may be reviewed for:
- Correctness.
- Reusability.
- Documentation quality.
- Alignment with the repository purpose.

## Community expectations

Please be respectful and constructive in all discussions. This project follows the Code of Conduct in `CODE_OF_CONDUCT.md`.

## Maintainer notes

Maintainers may request revisions, close duplicate issues, or decline changes that do not fit the project scope.
